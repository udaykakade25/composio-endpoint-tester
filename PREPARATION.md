## Required setup (do this now)

### 1. Composio account + API key

You'll use [Composio](https://composio.dev) to call API endpoints with managed OAuth auth.

1. Go to [platform.composio.dev](https://platform.composio.dev) and create a free account
2. Navigate to **Settings → API Keys** and generate an API key
3. Save it somewhere — you'll need it during setup

### 2. Connect your Google account

Your agent will test Gmail and Google Calendar endpoints using your own Google account. During setup, you'll be asked to connect your Gmail and Google Calendar via OAuth. To minimize issues:

- Use a **personal Google account** (not a workspace/org account — these sometimes have restricted OAuth)
- **Strongly recommended: use a throwaway or secondary account** — your agent will take real actions: sending emails, trashing messages, creating/deleting calendar events, etc.
- Make sure the account has some emails and calendar events (the agent needs real data to test against)

### 3. Loom account

You'll need to record a 2–4 minute video explaining your architecture at the end.

1. Go to [loom.com](https://loom.com) and create a free account
2. Install the Loom desktop app or browser extension
3. Do a quick test recording to make sure screen + mic capture works

### 4. AI coding agent

You'll use an AI coding agent to build your solution. We recommend:

- **[Codex CLI](https://github.com/openai/codex) with GPT-5.3-Codex** for implementation (fast and accurate for coding tasks)
- **[Claude Code](https://docs.anthropic.com/en/docs/claude-code) with Claude Opus 4.6** for high-level planning, architecture design, and writing your architecture doc

You're welcome to use whatever tools you prefer — Cursor, Windsurf, or any other agent. Use your own API keys / subscriptions.

**Can't afford API access?** If cost is a barrier, reach out to us at **prateek@composio.dev** or **pranjali@composio.dev** — we can provide API keys so you can focus on the assignment, not on billing.

### 5. Bun runtime

The project uses [Bun](https://bun.sh) (not Node.js).

```bash
# Install Bun if you don't have it
curl -fsSL https://bun.sh/install | bash
```

Verify it works: `bun --version`

## What to expect

- You'll get a set of API endpoint definitions (JSON) for Gmail and Google Calendar
- Your job is to build an **agent** that tries to execute each endpoint and classifies it as valid, invalid, insufficient scopes, or error
- Some endpoints are **fake** — they don't exist in the real API
- Some endpoints have **dependencies** — e.g., you need to list messages before you can get a specific message by ID
- Your agent should generalize to any app, not just Gmail/Calendar

## What you'll submit

1. **Your agent code** — implemented in `src/agent.ts`
2. **A test report** — `report.json` generated by running your agent
3. **An architecture doc** — `ARCHITECTURE.md` explaining your design decisions
4. **A Loom video** (2–4 minutes) — walking through your architecture and tradeoffs

## Tips

- **Don't rush into coding.** Spend a few minutes understanding the problem and planning your architecture. The architecture doc and video matter as much as the code.
- **Read `src/types.ts`** before you start — it defines the exact input/output contract.
- **Run `bun src/index.ts`** to see the endpoints you'll be testing.
- **Test early and often** with `bun src/run.ts` — it validates your output format.
- **Start with simple endpoints** (no path params, no body) to get your agent working, then handle the harder cases.
